# W7 — Cloud deploy + demo hardening (Dat's lane — needs account access)

**Branch:** `w7-deploy-demo` · ~40 min · makes it REAL and fixes local
tick lag (~6s locally vs 1.5s design — cloud scheduling is much tighter).
**Own:** `index.html` (title/meta only), NEW `docs/DEPLOY.md` (log what
you did), `docs/DEMO.md` (timing updates from rehearsal). Nothing else.

1. **Convex cloud:** `npx convex login` (browser OAuth) →
   `npx convex deployment select cloud` if prompted / `npx convex dev`
   once logged in creates the cloud dev deployment and pushes. Then set
   env on THE CLOUD deployment (they don't carry over from local):
   `npx convex env set RESPAN_API_KEY …`, `RESPAN_BASE_URL` (current
   working value: `https://api.openai.com/v1`; flip to Respan's gateway
   only after the dashboard provider is configured and smoke-tested),
   optionally `RESPAN_MODEL`. Verify: `npx convex run intel:smokeTest`.
2. **Frontend:** `npm i -g vercel` (or use the dashboard) →
   `vercel --prod` from the repo. Env vars in Vercel:
   `VITE_CONVEX_URL` = the cloud deployment URL (shown by convex dev /
   dashboard), `VITE_MAPBOX_TOKEN` = the pk. token. The build is
   `npm run build`, output `dist/` (Vite defaults — Vercel detects it).
3. `index.html`: `<title>LOCUS — SAR Command</title>` + a one-line meta
   description. Nothing else.
4. **Seed + rehearse on PROD:** open the Vercel URL → Open demo case →
   Run → Start scripted tips → full 90-second run against `docs/DEMO.md`,
   twice, with a phone timer. Update DEMO.md timings to what you measured
   (cloud tick cadence will differ from local).
5. **Record the disaster-fallback video** of the best rehearsal run
   (QuickTime screen recording is fine). Park the file OUTSIDE the repo
   (e.g. Desktop) — do not commit a video.
6. Check the found beat on prod, and that BOTH your laptop and a phone
   show the same live state (that's a pitch flex: "open it on your phone").

Accept: a public URL anyone in the room can open, running the cloud
deployment, with a rehearsed, timed, recorded demo. If any cloud step
stalls > 10 min, fall back to demoing localhost and say nothing — a
working local demo beats a broken deploy story.
