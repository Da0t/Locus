# W5 — After-action debrief (gateway)

**Branch:** `w5-debrief` · ~40 min · gives Respan a second distinct role.
**Own:** NEW `convex/debrief.ts`, `convex/sim.ts` (ONE line in the found
check), `src/components/FoundOverlay.tsx`. Nothing else.

Schema ready on main: `cases.debrief` (optional string).

1. `convex/debrief.ts` — `generate` internalAction({caseId}):
   - Gather via an internalQuery in this file: case facts, find location
     cell + sim clock at find, all tips (text/credibility/corroborates),
     final hypothesis weights + reasoning, count of searched cells.
   - ONE `generateText` call via the gateway (`respan.chat(MODEL)`, import
     from `./agents/client`): system = "You write terse SAR after-action
     debriefs. 5 short lines: OUTCOME / TIMELINE / DECISIVE INTEL / WHAT
     THE MODEL GOT RIGHT / ONE IMPROVEMENT." Feed the facts as JSON.
   - Patch `cases.debrief` via internalMutation. On LLM failure, write a
     plain template from the same facts — the panel must never be empty.
2. `convex/sim.ts` — in the found check (search "foundAtTick"), after the
   case is patched to "found", add exactly:
   `await ctx.scheduler.runAfter(0, internal.debrief.generate, { caseId });`
   Touch nothing else in the file.
3. `src/components/FoundOverlay.tsx` — when `activeCase.debrief` exists,
   render it under the SUBJECT LOCATED strip: mono, small, pre-line,
   amber headings. While undefined and status is "found", show
   "generating after-action debrief…" shimmer.

Accept: force a found (run the sim to completion, or temporarily seed
`hiddenTrue` next to the LKP in a scratch case) → within ~5s the overlay
shows a 5-line debrief naming the decisive tip. That's the closing beat
of the pitch.
