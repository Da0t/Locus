# W2 — Cron escalation sweep

**Branch:** `w2-cron-escalation` · ~30 min · Convex cron checklist item.
**Own:** NEW `convex/crons.ts`, NEW `convex/escalation.ts`. Nothing else.

1. `convex/escalation.ts` — `sweep` internalMutation, no args:
   - Load the active case (status "active"); bail if none or sim not
     running.
   - From `simState.heatmap` + the grids table: find cells with
     probability > 0.6 that are unsearched AND unclaimed. If any, take the
     hottest; convert to lat/lng via `cellCenter` (`./lib/geo`).
   - Insert a commands row (this is how alerts reach the console with ZERO
     UI work): `{ caseId, rawText: "[AUTO] escalation sweep",
     intent: { type: "query_status", question: "escalation" },
     status: "applied", response: "ESCALATION: grid (x,y) holds ~NN% of
     probability mass and is unassigned. Recommend tasking a team." }`
   - Dedupe: skip if the latest commands row is already an [AUTO] escalation
     for the same cell (read last ~5 rows).
2. `convex/crons.ts`:
   ```ts
   import { cronJobs } from "convex/server";
   import { internal } from "./_generated/api";
   const crons = cronJobs();
   crons.interval("escalation sweep", { minutes: 2 }, internal.escalation.sweep, {});
   export default crons;
   ```

Accept: pause all teams mentally (or just wait — planner keeps them busy,
so temporarily claim the hot cell's team elsewhere via the UI), within 2
min an ESCALATION line appears in the console log. No duplicate spam.
Pitch line: "a cron re-audits coverage every two minutes even if every
human is heads-down."
