# W1 — Tip corroboration (vector search + gateway embeddings)

**Branch:** `w1-vector-corroboration` · ~45 min · hits BOTH sponsors.
**Own:** NEW `convex/corroborate.ts`, `convex/tips.ts` (2-line hook),
`src/components/TipsFeed.tsx`. Nothing else.

Schema is ready on main: `tips.embedding` (optional float64[], 1536-dim
vector index `by_embedding`, filter `caseId`) and `tips.corroborates`
(optional tip id).

1. `convex/corroborate.ts` — `embedTip` internalAction({tipId, caseId}):
   - Load the tip (internalQuery in this file). Embed via the gateway:
     ```ts
     import { embed } from "ai";
     import { respan } from "./agents/client";
     const { embedding } = await embed({
       model: respan.textEmbedding("text-embedding-3-small"),
       value: tip.text,
     });
     ```
   - `ctx.vectorSearch("tips", "by_embedding", { vector: embedding,
     limit: 3, filter: (q) => q.eq("caseId", caseId) })` — take the best
     result that isn't this tip and scores > 0.80.
   - Patch via an internalMutation in this file: `{ embedding,
     corroborates: bestId ?? undefined }`; if corroborated, also bump the
     tip's credibility by +0.1 (cap 0.95) — corroborated intel is stronger.
2. `convex/tips.ts` — in `addTip`, after the `onNewTip` schedule, add:
   `await ctx.scheduler.runAfter(0, internal.corroborate.embedTip, {tipId, caseId})`.
3. `src/components/TipsFeed.tsx` — when a tip has `corroborates`, render a
   small `↳ corroborates earlier report` line (amber, font-mono, 9px) under
   the text. (Feed already gets full tip docs; find the referenced tip in
   the same query result by id for a snippet if cheap.)

Accept: add two similar tips ("saw her near the creek" twice, different
words) → second one shows the corroboration line and higher credibility;
a dissimilar tip does not. Embedding failure must be caught and logged —
tips must still work if the gateway is down.
